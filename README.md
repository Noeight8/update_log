# 更新日志插件 (Update Log)

一个为 AstrBot 设计的更新日志管理插件，支持全局公告和群专属公告的发布、查询、搜索与删除，并可自动推送到指定群聊。

## ✨ 功能特性

- **双模式公告**：支持**全局更新**（推送到所有白名单群）和**群专属更新**（仅推送到指定群）
- **独立ID管理**：全局与每个群的公告均有独立递增ID，便于定位
- **自动推送**：发布公告后可自动发送到目标群（可配置）
- **权限分离**：管理员可发布/删除公告，普通成员仅可查询/搜索
- **数据持久化**：所有公告存储于插件目录下的 `update_log.json` 文件
- **便捷查询**：支持按ID精确查询、关键词搜索（返回完整内容）

## ⚙️ 配置

在 AstrBot 的配置文件中（或通过 WebUI）设置以下字段：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `admin_ids` | `list[str]` | `[]` | 拥有管理权限的用户ID（字符串形式） |
| `global_whitelist` | `list[str]` | `[]` | 全局更新自动推送的目标群号列表 |
| `auto_push` | `bool` | `true` | 发布公告后是否自动发送到对应群 |
| `max_show_count` | `int` | `5` | （当前版本未使用）保留字段 |

> **注意**：`admin_ids` 和 `global_whitelist` 中的ID请填写为字符串，如 `["123456789"]`。

## 📖 指令列表

所有指令均以 `/log` 为前缀，支持指令组。

### 👑 管理员指令

| 指令 | 格式 | 说明 |
|------|------|------|
| 帮助 | `/log help` | 显示管理员完整帮助菜单 |
| 发布全局更新 | `/log global <内容>` | 发布一条全局公告，自动推送到 `global_whitelist` 中的所有群 |
| 发布群专属更新 | `/log group <群号> <内容>` | 为指定群发布专属公告，自动推送到该群 |
| 删除全局更新 | `/log del global <ID>` | 删除指定ID的全局公告，剩余公告ID自动重排 |
| 删除群专属更新 | `/log del group <群号> <ID>` | 删除指定群中指定ID的公告，剩余ID自动重排 |
| 查看统计 | `/log list` | 显示全局及各群存储的公告数量 |

### 👤 普通用户指令

| 指令 | 格式 | 说明 |
|------|------|------|
| 帮助 | `/log help` | 显示普通用户帮助菜单 |
| 查询全局公告 | `/log id global <ID>` | 根据ID查看全局公告的完整内容 |
| 查询本群公告 | `/log id group <ID>` | 根据ID查看**当前群**的专属公告（仅在群内有效） |
| 搜索公告 | `/log search <关键词>` | 在全局和所有群专属公告中搜索关键词，返回前5条完整内容 |

## 💾 数据存储

插件会在自身目录下创建 `update_log.json`，结构如下：

```json
{
  "global": [
    { "id": 1, "content": "全局更新内容", "time": "2025-01-01 12:00" }
  ],
  "group": {
    "123456789": [
      { "id": 1, "content": "群专属内容", "time": "2025-01-01 12:05" }
    ]
  }
}
```

全局与每个群的ID独立递增
删除操作后剩余公告会自动重新排序（ID连续）

## 📝 注意事项

1. **权限校验**：管理员 ID 需与 `admin_ids` 配置中的值**完全一致**（字符串比较）。不同平台返回的 ID 格式可能不同（例如 QQ 为纯数字字符串），请确保配置准确。
2. **群号格式**：`global_whitelist` 和 `/log group` 中的群号必须使用**字符串**，与机器人框架内部标识保持一致（例如 `"123456789"`）。
3. **自动推送**：当 `auto_push` 为 `true` 时，发布公告会尝试发送到目标群。若机器人未加入群或网络异常，错误信息会记录到日志，不影响公告数据存储。
4. **搜索限制**：关键词搜索默认返回最多 **5 条**完整公告，避免刷屏。如需调整，可修改代码中的 `len(res) < 5` 限制。
5. **群专属查询**：`/log id group` 指令**仅限群聊**使用，在私聊中无效。
6. **ID 重排**：删除公告后，剩余公告的 ID 会自动重新排序（从 1 开始连续递增），确保无空洞。
7. **数据安全**：所有数据存储在插件目录下的 `update_log.json` 文件中。建议定期备份，避免误删。
